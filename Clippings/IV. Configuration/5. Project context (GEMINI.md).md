# Project Context (GEMINI.md)

Gemini CLI uses `GEMINI.md` files to provide persistent, project-specific context to the AI agent. These files are the primary way to communicate team-shared conventions, architectural rules, and repo-wide workflows.

## Key Features

*   **Hierarchical Context:** `GEMINI.md` files can be placed at the root of a project or within subdirectories to provide scoped instructions.
*   **Team-Shared:** Unlike private memory, `GEMINI.md` files are intended to be committed to your version control system (e.g., Git) and shared with the entire team.
*   **Automatic Loading:** Gemini CLI automatically detects and loads these files into the agent's context at the start of a session.

## Usage Guidelines

1.  **Root Instructions:** Use `./GEMINI.md` for repo-wide guidance, such as coding standards, preferred libraries, and build commands.
2.  **Subdirectory Instructions:** Use `./src/GEMINI.md` (or similar) for instructions specific to a module or component.
3.  **Memory v2 Integration:** When `experimental.memoryV2` is enabled, the agent can directly update these files to persist newly learned project conventions.
