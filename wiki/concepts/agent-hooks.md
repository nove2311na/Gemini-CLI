# Agent Hooks

Agent hooks intercept the agent's execution flow at two critical points: before planning begins and after responses are generated. These hooks enable prompt validation, context injection, and response quality control.

## BeforeAgent

Fires after a user submits a prompt but before the agent begins planning. This is the earliest point in the turn lifecycle where external scripts can intervene.

### Available Input Fields

| Field | Type | Description |
|-------|------|-------------|
| `prompt` | string | Original text submitted by the user |

### Response Options

- **hookSpecificOutput.additionalContext**: Text appended to prompt for this turn only
- **decision: "deny"**: Blocks turn and discards user's message (not saved to history)
- **continue: false**: Blocks turn but saves message to history
- **reason**: Required when denying or stopping

### Exit Code 2 Behavior

Aborts the turn and erases the prompt from context, equivalent to `decision: "deny"`.

### Example Use Cases

1. **Prompt Validation**: Check if user requests are appropriate or in scope
2. **Dynamic Context Injection**: Load relevant files or data based on prompt content
3. **Rate Limiting**: Track request frequency and block excessive usage
4. **Template Injection**: Prepend structured templates to standard request patterns
5. **Preprocessing**: Rewrite or normalize user input before agent sees it

## AfterAgent

Fires once per turn after the model generates its final response. Primary use case is response validation and automatic retry handling.

### Available Input Fields

| Field | Type | Description |
|-------|------|-------------|
| `prompt` | string | User's original request |
| `prompt_response` | string | Final text generated by the agent |
| `stop_hook_active` | boolean | Indicates if hook is already running as part of retry sequence |

### Response Options

- **decision: "deny"**: Rejects response and forces retry with `reason` as new prompt
- **hookSpecificOutput.clearContext**: Clears conversation history while preserving UI display
- **continue: false**: Stops session without retrying
- **reason**: Required when denying or stopping

### Exit Code 2 Behavior

Rejects response and triggers automatic retry turn using stderr as the feedback prompt.

### Example Use Cases

1. **Response Validation**: Check for policy violations or inappropriate content
2. **Quality Gates**: Verify response meets formatting or completeness requirements
3. **Automatic Corrections**: Provide hints for common errors or style issues
4. **Context Reset**: Clear conversation history for privacy between topics
5. **Structured Output Enforcement**: Ensure responses conform to expected schemas

## Comparison: BeforeAgent vs AfterAgent

| Aspect | BeforeAgent | AfterAgent |
|--------|-------------|------------|
| Timing | Before planning | After response generation |
| Input | User prompt only | Prompt + generated response |
| Denial Effect | Discards or saves prompt | Forces retry |
| Exit Code 2 | Erases from context | Triggers retry |
| Use Case Priority | Validation, injection | Quality control, correction |

## Integration with Model Hooks

Agent hooks operate at a higher abstraction level than [[model-hooks]]. While model hooks work directly with LLM request/response structures, agent hooks work with the full turn context including user interaction patterns.

Best practices:
- Use BeforeAgent for structural changes (context injection, prompt rewriting)
- Use AfterAgent for outcome-based decisions (validation, retry logic)
- Consider combining with BeforeModel for LLM-level modifications
- Avoid conflicting modifications across hook types
