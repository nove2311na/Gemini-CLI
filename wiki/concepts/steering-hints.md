# Steering Hints

Steering hints are user-submitted text feedback provided during active [[model-steering]] execution. When model steering is enabled, any text typed while the agent is working becomes a steering hint that influences the agent's next actions.

## Definition

A steering hint is freeform text input from the user that is sent to [[gemini-cli]] while the agent spinner is visible. Unlike standard prompts, steering hints are not sent as new conversation turns but are instead injected as guidance during active task execution.

## How They Work

1. User initiates a task (for example, "Refactor the database service")
2. While the agent is working with the spinner visible, the user types feedback
3. Pressing **Enter** submits the text as a steering hint
4. Gemini CLI acknowledges the hint with a brief message (generated by a small, fast model)
5. The hint is injected into the model's context for the very next turn
6. The model re-evaluates its current plan and adjusts actions accordingly

## Types of Steering Hints

Based on the official documentation, steering hints can be categorized by their purpose:

| Type | Example | Purpose |
|------|---------|---------|
| Path correction | "The utilities are in `src/common/utils`." | Redirect to correct location |
| Step skipping | "Skip the unit tests for now." | Remove tasks from current plan |
| Context addition | "The `User` type is in `packages/core/types.ts`." | Provide missing information |
| Effort redirecting | "Stop searching and start drafting." | Change current action |
| Ambiguity resolution | "Use the existing `Logger` class." | Guide decision-making |

## Relationship to Context Injection

Steering hints are the user-facing interface for [[context-injection]]. The technical mechanism prepends an internal instruction to each steering hint that tells the main agent to re-evaluate its active plan, classify the update, and apply minimal-diff changes.

## Trigger Condition

Steering hints only work when the agent is actively executing a task — specifically when the spinner is visible. This means users cannot preemptively steer the agent before it starts working.
