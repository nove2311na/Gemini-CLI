# Custom Commands

This source document from geminicli.com documents the custom commands feature in Gemini CLI, which allows users to save and reuse frequently used prompts as personal shortcuts. The document covers file locations, TOML format specification, argument handling methods, and injection syntax for creating dynamic, reusable commands.

## Summary

Custom commands enable workflow automation through saved prompt templates that can be invoked globally or on a per-project basis. The feature uses TOML files stored in either the user commands directory (`~/.gemini/commands/`) or project commands directory (`<project>/.gemini/commands/`). Project commands take precedence over global commands with the same name, allowing teams to customize behavior while maintaining global defaults.

## Key Concepts Covered

- **Command precedence**: Project commands override global commands
- **Namespacing**: Subdirectories create namespaced commands using `:` separator
- **Argument injection**: Two modes—explicit `{{args}}` placeholder and default handling
- **Shell injection**: `!{...}` syntax embeds shell command output in prompts
- **File injection**: `@{...}` syntax embeds file or directory content

## Practical Applications

- Creating reusable refactoring commands (`/refactor:pure`)
- Git workflow automation (`/git:commit`, `/git:fix`)
- Code review templates with injected context
- Changelog generation with structured prompts
