# Gemini CLI Settings Reference

This source documents the comprehensive settings system for Gemini CLI, providing a complete reference for all configuration options accessible via the `/settings` command and `settings.json` file.

## Settings Structure

Settings are stored in a hierarchical structure with workspace settings overriding user-level settings:

- **User settings**: `~/.gemini/settings.json`
- **Workspace settings**: `your-project/.gemini/settings.json`

The `/settings` command provides an interactive dialog for viewing and editing all settings, mirroring the JSON file structure.

## Key Settings Categories

The reference covers 14 distinct categories spanning general behavior, output formatting, UI customization, model configuration, security, and experimental features.

### Approval Modes

The `general.defaultApprovalMode` setting controls tool execution authorization:

- `"default"` — Prompts for approval before tool execution
- `"auto_edit"` — Auto-approves edit tools only
- `"plan"` — Read-only mode, no tool execution

YOLO mode (auto-approve all actions) is intentionally restricted to CLI flags only (`--yolo` or `--approval-mode=yolo`), not accessible through settings.

### Session Management

Automatic session cleanup is configurable via `general.sessionRetention.*` settings:

- `general.sessionRetention.enabled` — Toggle for automatic cleanup (default: `true`)
- `general.sessionRetention.maxAge` — Retention period, e.g., `"30d"`, `"7d"`, `"1w"`

### Security Features

Tool sandboxing operates at granular levels:

- `security.toolSandboxing` — Tool-level isolation (default: `false`)
- `tools.sandboxAllowedPaths` — Whitelisted filesystem paths
- `tools.sandboxNetworkAccess` — Network access within sandbox (default: `false`)

CONSECA (`security.enableConseca`) enables LLM-generated dynamic security policies based on prompt context.

### Experimental Features

Notable experimental settings include:

- `experimental.modelSteering` — User hints during tool execution (default: `false`)
- `experimental.memoryV2` — Direct markdown file editing for context persistence (default: `true`)
- `experimental.autoMemory` — Background skill extraction from past sessions (default: `false`)
- `experimental.worktrees` — Git worktree management for parallel work (default: `false`)
