# ACP Mode

**ACP Mode** is a special operational mode of [[gemini-cli]] designed for programmatic control, primarily enabling IDE and developer tool integrations. It uses JSON-RPC over stdio to communicate between the Gemini CLI agent and external clients.

## Starting ACP Mode

To start Gemini CLI in ACP mode, use the `--acp` flag:

```bash
gemini --acp
```

## Core Architecture

ACP mode establishes a client-server relationship where:

- **Communication**: All traffic uses JSON-RPC 2.0 over standard input/output
- **Client role**: Sends requests (prompts) and handles responses from Gemini CLI
- **Server role**: Listens for JSON-RPC requests, processes them, and returns responses

The core implementation is located in `packages/cli/src/acp/acpClient.ts`.

## MCP Integration

ACP can be extended with the [[mcp-model-context-protocol]]. This allows ACP clients (such as IDEs) to expose their own functionality as tools the Gemini model can use:

1. Client implements an MCP server advertising its tools
2. During ACP `initialize` handshake, client provides MCP server connection details
3. Gemini CLI connects to the MCP server and discovers available tools
4. The model can then call these IDE tools through the agent

MCP client logic is in `packages/core/src/tools/mcp-client.ts`.

## Supported Methods

### Core Methods
- `initialize`: Establishes connection, registers MCP server
- `authenticate`: Authenticates the user
- `newSession`: Starts a new chat session
- `loadSession`: Loads a previous session
- `prompt`: Sends a prompt to the agent
- `cancel`: Cancels an ongoing prompt

### Session Control
- `setSessionMode`: Changes approval level for tool calls
- `unstable_setSessionModel`: Changes the model for current session

## Security: File System Proxy

ACP includes a proxied filesystem service. When the agent needs to read or write files, it does so through the ACP client. This ensures the agent only accesses files explicitly allowed by the user. See [[acp-file-system-proxy]] for details.

## Debugging and Telemetry

Enable debugging with the `--debug` flag:

```bash
gemini --acp --debug
```

For detailed telemetry, configure environment variables:

- `GEMINI_TELEMETRY_ENABLED=true`
- `GEMINI_TELEMETRY_TARGET=local`
- `GEMINI_TELEMETRY_OUTFILE=/path/to/your/log.json`

See [[acp-telemetry]] for full details.
