# Issue and PR Automation

This source documents the automated tools used by Gemini CLI to manage the contribution workflow, including PR review automation and self-service issue assignment.

## Automated Review Tool

The automated review tool is implemented as a shell script at `./scripts/review.sh` that accepts a PR number and an optional model parameter. This tool scans pull requests for common anti-patterns and testing issues, helping maintain contribution standards across the project.

## Issue Command Protocol

Two slash commands handle self-service issue management for community contributors:

- **`/assign`**: Adds a comment to self-assign the user to an issue. This command is gated by two requirements: the issue must have the "help wanted" label, and users may have a maximum of 3 active assignments at any time.

- **`/unassign`**: Adds a comment to self-unassign the user from an issue they previously claimed.

These commands implement a lightweight community workflow system that reduces maintainer overhead while still ensuring appropriate labeling and workload limits.

## Key Takeaways

The automation documented here represents a hybrid approach combining technical tooling (review script) with community process automation (issue commands). The review tool focuses on code quality enforcement, while the issue commands implement a self-service contribution management system with built-in guardrails.
