# Themes

**Source:** [Gemini CLI Documentation - Themes](https://geminicli.com/docs/cli/themes/)

## Summary

Gemini CLI provides a comprehensive theme system that allows users to customize the CLI's color scheme and appearance. The system includes 20+ pre-defined themes (10 dark, 7 light + ANSI variations) and supports custom themes defined in `settings.json`. Theme selection is accessible via the `/theme` command with arrow-key navigation, and selected themes persist automatically across sessions.

## Key Features

- **Pre-defined themes**: Built-in options including dark themes (Atom One, Dracula, Tokyo Night, etc.) and light themes (Ayu Light, Solarized Light, Xcode, etc.)
- **Custom themes**: Define personal themes via the `customThemes` block in `settings.json`
- **File-based themes**: Load themes from external JSON files (restricted to home directory for security)
- **Extension themes**: Extensions automatically contribute their themes to the selection list
- **Theme persistence**: Preferences are remembered across sessions

## Important Notes

### Configuration Conflict

If a theme is defined in `settings.json` (either by name or file path), the `/theme` command cannot be used to change it. Users must manually edit the config file to reset.

### Security Constraint

Theme files can only be loaded from within the home directory for security reasons.

## Configuration Schema

Custom themes support five configuration objects:

- **text**: `primary`, `secondary`, `link`, `accent`, `response`
- **background**: `primary`, `diff.added`, `diff.removed`
- **border**: `default`, `focused`
- **status**: `success`, `warning`, `error`
- **ui**: `comment`, `symbol`, `gradient`
