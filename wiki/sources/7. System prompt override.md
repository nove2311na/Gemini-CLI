# System Prompt Override (GEMINI_SYSTEM_MD)

The core system instructions that guide Gemini CLI can be completely replaced with an external Markdown file. This feature is controlled via the `GEMINI_SYSTEM_MD` environment variable.

## Overview

The `GEMINI_SYSTEM_MD` variable instructs the CLI to use an external Markdown file for its system prompt, completely overriding the built-in default. This is a **full replacement, not a merge** — if you use a custom file, none of the original core instructions will apply unless you explicitly include them.

This feature is intended for advanced users who need to enforce strict, project-specific behavior or create a customized persona.

## Key Features

- **Full replacement model**: Custom prompts completely override built-in instructions
- **Three path modes**: Project default (`.gemini/system.md`), custom path, or disabled
- **Variable substitution**: Dynamically inject built-in content via placeholders
- **Visual indicator**: `|⌐■_■|` signals custom system-prompt mode in the UI
- **Export workflow**: `GEMINI_WRITE_SYSTEM_MD` exports defaults for review before modification

## Path Modes

| Mode | Value | Behavior |
|------|-------|----------|
| Project default | `GEMINI_SYSTEM_MD=true` or `GEMINI_SYSTEM_MD=1` | Reads `.gemini/system.md` relative to project |
| Custom path | `GEMINI_SYSTEM_MD=/path/to/file.md` | Absolute/relative path with tilde expansion |
| Disabled | `GEMINI_SYSTEM_MD=false` or `0` | Uses built-in prompt |

## Variable Substitution

The following placeholders inject built-in content into custom prompts:

- `${AgentSkills}` — Complete section of all available agent skills
- `${SubAgents}` — Complete section of available sub-agents
- `${AvailableTools}` — Bulleted list of all enabled tool names
- `${toolName_ToolName}` — Individual tool names (e.g., `${write_file_ToolName}`)

## Recommended Workflow

1. Export the current default: `GEMINI_WRITE_SYSTEM_MD=1 gemini`
2. Review the exported file to understand required safety rules
3. Selectively modify or replace as needed
4. Enable override via `.gemini/.env` or shell export

## system.md vs GEMINI.md

| File | Purpose | Nature |
|------|---------|--------|
| `system.md` | Operational rules, safety, tool protocols | Firmware-like, stable |
| `GEMINI.md` | Persona, goals, methodology, project context | Strategy, evolves per task |
