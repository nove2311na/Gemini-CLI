# Project Context (GEMINI.md)

This documentation page from geminicli.com describes how Gemini CLI uses `GEMINI.md` files to provide persistent, project-specific context to the AI agent. These files serve as the primary mechanism for communicating team-shared conventions, architectural rules, and repo-wide workflows.

## Key Features

The documentation highlights several core capabilities of the GEMINI.md system. **Hierarchical Context** allows configuration files to be placed at any directory level, enabling scoped instructions for specific modules or components. **Team-Shared** design means these files are intended to be committed to version control and shared across the entire team, distinguishing them from private notes. **Automatic Loading** ensures that Gemini CLI detects and loads these files into the agent's context at the start of each session without manual invocation.

## Usage Guidelines

The page provides guidance on placing instructions at different levels. Root-level `./GEMINI.md` files provide repo-wide guidance such as coding standards, preferred libraries, and build commands. Subdirectory files like `./src/GEMINI.md` contain instructions specific to individual modules. The documentation also covers **Memory v2 Integration**, noting that when the `experimental.memoryV2` flag is enabled, the agent can directly update these files to persist newly learned project conventions.

## Significance

This source establishes GEMINI.md as a foundational feature for team-based AI agent workflows. By enabling persistent, version-controlled configuration, it transforms Gemini CLI from a purely interactive tool into one that can enforce team standards and architectural decisions consistently across all contributors.
