# NPM Package Structure

This source document from the official Gemini CLI documentation describes the monorepo architecture of the project, which consists of two main packages: [[@google/gemini-cli]] and [[@google/gemini-cli-core]]. The document explains how these packages are structured, published, and managed using NPM workspaces.

## Key Points

The project uses a **two-package architecture** that separates user-facing functionality from core API logic. This separation enables the [[@google/gemini-cli-core]] package to be consumed independently as a standard Node.js package, while the main CLI package is bundled into a single executable for distribution.

The monorepo is managed using [[npm-workspaces]], which provides simplified dependency management, automatic symlinking of internal packages, and streamlined script execution through the `--workspace` flag.

## Architecture Summary

| Package | Purpose | Distribution Method |
|---------|---------|---------------------|
| [[@google/gemini-cli]] | UI, command parsing, user-facing features | Bundled into single executable |
| [[@google/gemini-cli-core]] | API requests, authentication, caching | Standard Node.js package with dependencies |

## Practical Utilities

Developers contributing to or extending the project can leverage workspace commands to run scripts on specific packages:

```bash
npm run build --workspace @google/gemini-cli
npm run test --workspace @google/gemini-cli-core
```
