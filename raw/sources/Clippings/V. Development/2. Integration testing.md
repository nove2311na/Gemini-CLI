# Integration testing

Integration tests are designed to validate the end-to-end functionality of the Gemini CLI. They are not run as part of the default `npm run test` command.

To run the integration tests, use the following command:

```bash
npm run test:e2e
```

### Forking and Secrets

If you are forking the repository, in order to make the integration tests run you'll need to add a GitHub Repository Secret with a value of `GEMINI_API_KEY` and set that to a valid API key.
